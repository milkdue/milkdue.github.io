基本类型

  1. typescript 的基本数据类型包括 布尔类型、数值类型、字符串类型、数组类型、元组类型、枚举类型…
  2. 类型注释，当使用 const var let 等变量声明时可以添加类型注释来显式指定变量的类型，但是在大多数情况下，这不是必须的，TypeScript 会尝试自动推断代码中的类型。例如，变量的类型根据其初始化器的类型推断出来

undefined和null

  1. ts 中 undefined 和 null 即是类型也是值

never类型

  1. never 类型表示永远不存在的类型
  2. never 类型是任意类型的子类型
    • never 类型的值可以赋值给任意类型
  3. 没有任何类型是 never 类型的子类型
    • 任何类型的值都不能赋值给 never 类型

类型断言

  1. 使用类型断言来指定更具体的类型
    • 与类型注释一样，类型断言由编译器删除，不会影响代码运行时的行为
    • 类型断言在编译时被移除，所以没有与类型断言相关联的运行时检查，null 如果断言错误，不会出现异常
    • ts 只允许类型断言转换为更具体或不太具体的类型版本，此规则可防止“不可能”的强制 const x = 'hello' as number;
  2. 类型断言的两种写法
    • as 关键字 const x = 'hello' as any;
    • <> <any> x jsx 不可以用 <> 只能使用 as

symbol

  1. 用来表示独一无二的值
  2. ts 中使用首先引入 lib (对应的库)(如下：tsconfig.json)
  3. 将 symbol 作为对象的属性名
  4. 属性名的遍历，下面是一些属性名的遍历方式
    • for in 无法遍历 symbol 作为属性名的属性
    • Object.keys(object) 无法遍历 symbol 作为属性名的属性 返回属性名组成的数组
    • Object.getOwnPropertyNames(object) 无法遍历 symbol 作为属性名的属性 返回属性名组成的数组
    • JSON.stringfy(object) 无法出现 symbol 作为属性名的属性
    • Object.getOwnPropertySymbols(object) 只会出现 symbol 作为属性名的属性 返回属性名组成的数组
    • Reflect.ownKeys(object) 返回所有属性名返回的数组包括 symbol 作为属性名的数组
  5. Symbol 的两个静态方法
    • Symbol.for(): 全局创建一个 symbol
    • Symbol.keyFor(symbol): 返回使用 Symbol.for 创建的 symbol 的标识
  6. 11 个内置的 symbol 值
    • Symbol.hasInstance: 使用 instanceof 判断是否是某对象的实例时调用的内置的方法
    • Symbol.isConcatSpreadable: 一个可读写的布尔值，当一个数组的该属性是 false 时，使用 concat 方法这个数组就不会被扁平化
    • Symbol.species: 一个函数值属性，其被构造函数用以创建派生对象
